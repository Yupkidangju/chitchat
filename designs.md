# designs.md

## 1. 개요

`chitchat`의 UI/UX 방향은 **Clean Neo-Brutal Desktop Workspace**로 고정한다. 핵심은 강한 검은 외곽선, 평면적인 카드, 명확한 단계 표시, 과장 없는 색상 대비를 사용해 Provider 설정부터 프롬프트 조립·채팅 실행·Inspector 검증까지 한 번에 이해되는 데스크톱 앱을 만드는 것이다.

이 디자인은 장식적인 네오-브루탈리즘이 아니라 구현자가 바로 만들 수 있는 **구조적 네오-브루탈리즘**이다. 모든 주요 영역은 두꺼운 선, 명확한 제목 바, 강한 CTA, 단일 선택 상태, 상태 배너, 고정된 폼 검증 규칙으로 닫는다.

핵심 화면은 다음 흐름을 따른다.

```txt
Provider 등록 → Key 저장/연결 테스트 → 모델 목록 불러오기 → ModelProfile 저장
→ UserPersona / AIPersona / Lorebook / Worldbook 작성
→ ChatProfile 조합 → PromptOrder 확인 → New Chat → Send → Prompt Inspector 검증
```

## 2. 제품 컨텍스트

### 제품

- 제품명: `chitchat`
- 제품 유형: Python FastAPI 백엔드 + HTML/CSS/JS 웹 프론트엔드 AI 채팅 앱
- 백엔드: FastAPI + uvicorn
- 프론트엔드: 순수 HTML/CSS/JS SPA (Neo-Brutal 디자인 시스템)
- 저장소: SQLite 로컬 파일
- Secret 저장: OS keyring, DB에는 `secret_ref`만 저장
- 지원 Provider: Gemini, OpenRouter, LM Studio
- 캐릭터 시스템: VibeSmith 9섹션 동적 페르소나 + ZSTD 압축 상태 저장

### 대상 사용자

- LLM Provider와 모델 파라미터를 직접 관리하는 고급 사용자
- 창작용 AI 채팅을 위해 페르소나, 로어북, 세계관, 프롬프트 순서를 조합하는 사용자
- 프롬프트가 어떻게 구성됐는지 실제 요청 직전에 검증하려는 사용자

### 비즈니스 목표

MVP의 목표는 사용자가 외부 기획 회의 없이 로컬 데스크톱에서 Provider 등록, 모델 설정, 프로필 조합, 프롬프트 검증, 스트리밍 채팅까지 수행하게 하는 것이다.

### 제약 조건

- UI는 Provider Adapter를 직접 호출하지 않는다. UI는 ViewModel과 Service만 호출한다.
- API Key는 SQLite에 노출하지 않는다.
- 모델 capability에 없는 파라미터 컨트롤은 화면에서 숨긴다.
- PromptOrder는 사용자가 재정렬할 수 있어야 한다.
- Inspector는 조합된 PromptBlock, 로어북 매칭 결과, token estimate, 잘린 history ID를 보여준다.
- RisuAI의 코드, UI, 에셋, 문구, 스키마를 복제하지 않는다.

### 가정

- v0.1 BETA는 단일 라이트 테마만 제공한다.
- 최소 지원 창 크기는 `1120 x 720`이다.
- 권장 창 크기는 `1440 x 900`이다.
- 960px 미만 폭에서는 탐색과 Inspector를 접는 compact desktop layout을 사용한다.
- 모바일 앱은 v0.1 BETA 타깃이 아니지만, 좁은 화면 동작은 향후 이식이 가능하도록 정의한다.

## 3. 디자인 원칙

1. **설정의 명확성이 시각적 장식보다 우선**  
   Provider 등록과 모델 capability 상태가 가장 먼저 이해되어야 한다. 장식보다 단계, 상태, CTA 조건을 우선한다.

2. **하나의 워크스페이스, 여러 편집기**  
   모든 설정 화면은 `목록 패널 + 편집 패널 + 상태/검증 패널` 패턴을 반복한다. 사용자가 새 화면마다 구조를 다시 학습하지 않게 한다.

3. **계약의 시각화**  
   숨겨진 데이터 계약은 UI에 표시한다. 예: `secret_ref`, `supported_parameters`, `max_output_tokens`, `matched_lore_entry_ids`, `truncated_history_message_ids`.

4. **강한 경계, 낮은 장식**  
   네오-브루탈리즘은 두꺼운 선과 명확한 면 분리로 구현한다. 과한 회전, 3D, 글리치, 강한 그림자는 쓰지 않는다.

5. **분위기보다 상태**  
   성공, 경고, 실패, streaming, stopped, archived 상태가 색상·문구·아이콘·버튼 활성 조건으로 동시에 드러나야 한다.

6. **프롬프트 신뢰가 제품이다**  
   채팅 화면의 우측 Inspector는 부가 기능이 아니라 신뢰의 핵심이다. 사용자는 응답 전후에 프롬프트 조합 근거를 확인할 수 있어야 한다.

### 내비게이션 계층

| 계층 | 영역 | 용도 |
|---|---|---|
| 앱 셸 | 사이드바 + 상단 상태 바 | 현재 화면, Provider 상태, DB 경로, 전역 알림 표시 |
| 주요 페이지 | 채팅, 공급자, 모델, 프로필 | 핵심 작업 수행 |
| 상세 패널 | 편집기, Inspector, 미리보기 | 선택 항목 편집 및 검증 |
| 오버레이 | 확인, 프롬프트 스냅샷, 오류 상세 | 흐름을 끊지 않고 상세 확인 |

### 기본 랜딩

앱 첫 실행 시 `채팅` 페이지를 열되, 채팅 시작 조건이 충족되지 않으면 중앙에 **셋업 체크리스트 빈 상태**를 표시한다.

체크리스트 순서:

1. Provider 저장
2. API Key 저장 또는 LM Studio endpoint 확인
3. 연결 테스트 성공
4. 모델 목록 불러오기
5. ModelProfile 저장
6. UserPersona 저장
7. AIPersona 저장
8. ChatProfile 저장
9. 새 채팅 생성

## 5. 시각적 방향

### 스타일 계열

- 기반: Neo-Brutalism
- 실행: 깔끔한 유틸리티 데스크톱 UI
- 보조 영향: AI 네이티브 워크스페이스, 기술 편집기, 프롬프트 디버깅 콘솔

### 시각 규칙

- 외곽 테두리는 `2px` 또는 `3px` 솔리드 잉크를 사용한다.
- 메인 패널은 그라디언트 없이 평면 표면을 사용한다.
- 강조 블록은 CTA, 활성 내비게이션, 경고, 선택된 카드에만 채도 높은 색상을 사용한다.
- 그림자는 오프셋 하드 섀도우이며, 흐림 유리 효과 그림자는 사용하지 않는다.
- 모서리는 약하게 둥글린다: 컨트롤 `6px`, 패널 `10px`.
- 아이콘은 `2px` 스트로크의 단일 SVG 아웃라인 계열을 사용한다.
- 밀집 편집 화면에서도 텍스트는 `12px` 이하로 내려가지 않는다.

### 표면 모델

| 표면 | 용도 | 스타일 |
|---|---|---|
| 앱 배경 | 전역 필드 | 따뜻한 오프화이트 |
| 기본 패널 | 페이지 콘텐츠 | 화이트, 3px 잉크 테두리 |
| 보조 패널 | 목록과 사이드 패널 | 크림, 2px 잉크 테두리 |
| 부상 액션 카드 | 빈 상태 및 셋업 단계 | 강조 배경, 3px 잉크 테두리, 4px 하드 섀도우 |
| Inspector 패널 | 프롬프트 검증 | 연한 시안 표면, 3px 잉크 테두리 |
| 오류 패널 | 실패 상세 | 연한 빨강 표면, 3px 잉크 테두리 |

### 디자인 금지 사항

- 보라색 AI 그라디언트 배경 금지.
- v0.1 BETA에서 다크 기본 테마 금지.
- 글라스모피즘 금지.
- 저대비 회색-위-회색 폼 금지.
- 파괴적이거나 상태 변경 작업에 아이콘만 사용하는 숨겨진 CTA 금지.

| `color.focus` | `#00D1FF` | 키보드 포커스 링 |

### 역할 규칙

- 주요 CTA는 오렌지 `#FF5A1F`에 검정 텍스트와 `3px` 테두리를 사용한다.
- 파괴적 작업은 기본적으로 빨강 채움이 아닌 화이트 표면에 빨강 테두리와 빨강 텍스트를 사용한다.
- 활성 내비게이션은 노란색 채움에 검정 텍스트를 사용한다.
- Inspector는 시안 틴트로 검증을 편집과 분리한다.
- 성공 상태는 색상만 사용하지 않는다: 체크 아이콘과 텍스트를 포함한다.
- 오류 상태는 색상만 사용하지 않는다: 오류 아이콘, 실패한 작업명, 복구 안내를 포함한다.

### 대비 규칙

- 모든 밝은 표면의 본문 텍스트는 `#111111`을 사용한다.
- 회색 텍스트는 필수 필드 라벨에 절대 사용하지 않는다.
- 비활성 컨트롤은 가시적인 라벨 대비를 유지하고 비활성 사유를 툴팁이나 도우미 텍스트로 표시한다.
- 토큰 예산 경고는 80%에서 경고, 95%에서 오류로 전환된다.

## 8. 레이아웃 및 그리드

### 앱 셸

데스크톱 레이아웃은 전체 높이 셸을 사용한다.

```txt
Window
├─ 사이드바: 236px 고정
├─ 메인 영역
│  ├─ 상단 상태 바: 48px 고정
│  └─ 페이지 콘텐츠: 유동적
└─ 전역 토스트 스택: 우상단, 최대 3개 표시
```

### 간격 리듬

| 토큰 | 값 | 용도 |
|---|---:|---|
| `space.1` | 4px | 밀집 아이콘 간격 |
| `space.2` | 8px | 라벨/컨트롤 간격 |
| `space.3` | 12px | 폼 행 간격 |
| `space.4` | 16px | 카드 패딩 |
| `space.5` | 20px | 패널 내부 간격 |
| `space.6` | 24px | 페이지 섹션 간격 |
| `space.8` | 32px | 대형 페이지 패딩 |

### 페이지 컨테이너

- 페이지 패딩: 데스크톱 `24px`, 컴팩트 `16px`.
- 패널 간격: `16px`.
- 크기 조절이 유용한 곳에서 메인 패널은 리사이저블 분할을 사용한다.
- 목록은 최소 너비 `240px`, 최대 너비 `340px`을 유지한다.
- Inspector는 데스크톱에서 `360px` 너비를 유지하고, `1280px` 미만에서는 우측 서랍으로 전환된다.

### 브레이크포인트

| 너비 | 동작 |
|---:|---|
| `>= 1440px` | 전체 3컬럼 워크스페이스; Inspector 토글 시 표시 |
| `1120px–1439px` | 사이드바 전체 표시; Inspector 서랍이 페이지 콘텐츠 위에 오버레이 |
| `960px–1119px` | 사이드바 72px 아이콘으로 축소; 페이지 2컬럼 분할 |
| `< 960px` | 컴팩트 데스크톱 모드; 내비게이션 상단 스트립; 편집기 수직 스택 |

### 모바일 축소 논리

모바일은 v0.1 대상이 아니다. 향후 좁은 화면 렌더링 시, 각 페이지는 다음 순서로 축소된다:

1. 페이지 헤더 및 주요 CTA
2. 목록 또는 선택기
3. 편집 폼
4. 미리보기 / Inspector
5. 보조 메타데이터

좁은 모드에서 테이블은 가로 스크롤을 유지하지 않는다. 테이블은 라벨이 있는 행으로 변환된다.

## 9. 핵심 컴포넌트

### 9.1 앱 사이드바

- 너비: `236px`, 축소 시 `72px`.
- 배경: `color.surface_alt`.
- 테두리: 오른쪽 `3px color.ink`.
- 활성 항목: 노란색 채움, 검정 테두리, 좌측 노치.
- 각 내비 항목: 아이콘 `20px`, 라벨 `14px/800`, 높이 `44px`.
- 비활성 내비 항목은 사용하지 않는다. 페이지는 항상 접근 가능하며 빈 상태를 표시한다.

### 9.2 상단 상태 바

표시 항목:

- 현재 페이지 제목
- 활성 Provider 상태 요약
- 모델 캐시 최신성
- SQLite 경로 축약 라벨
- 전역 오류 표시기

상태:

| 상태 | 표시 |
|---|---|
| 정상 | 녹색 칩 `연결됨` + Provider 이름 |
| 미테스트 | 노란색 칩 `테스트 안 됨` |
| 실패 | 빨간색 칩 `연결 실패` + 상세 버튼 |
| 스트리밍 | 파란색 칩 `스트리밍 중` + 세션 제목 |

### 9.3 버튼

| 변형 | 스타일 | 용도 |
|---|---|---|
| 주요 | 오렌지 채움, 검정 테두리, 하드 섀도우 | 저장, 전송, 새 채팅 |
| 보조 | 화이트 채움, 검정 테두리 | 엔트리 추가, 프롬프트 보기, 모델 가져오기 |
| 3차 | 투명, 호버 시 밑줄 | 상세 보기, ID 복사 |
| 위험 | 화이트 채움, 빨강 텍스트/테두리 | 보관, 초안 삭제 |
| 비활성 | 회색 채움, 검정 테두리 40% 불투명도 | 유효하지 않은 작업 |

버튼 상태:

- 호버: 섀도우 오프셋을 `2px`에서 `4px`로 변경하여 시각적 강조; 레이아웃 위치는 변경하지 않음.
- 프레스: 섀도우 오프셋 `0px`; 배경 4% 어두워짐.
- 포커스: 검정 테두리 바깥에 `2px` 시안 외부 링.
- 로딩: 좌측 스피너 + 동사 텍스트, 예: `테스트 중...`.

### 9.4 입력 및 폼

- 라벨은 항상 컨트롤 위에 표시한다.
- 필수 라벨 접미사: 빨간색 `*`.
- 테두리: `2px color.ink`.
- 포커스: 테두리 바깥에 `2px color.focus`.
- 유효하지 않음: 빨강 테두리 + 필드 아래 인라인 오류.
- 도우미 텍스트: 회색, 12px, 최대 한 문장.
- 최대 길이가 있는 필드에 문자 카운터를 표시한다.
- 비밀번호/API 키 필드는 마스킹된 값을 표시하고 `키 저장`을 별도 CTA로 제공한다.

### 9.5 카드

- 기본 카드: 화이트 표면, `2px` 테두리, `8px` 반경.
- 중요 카드: `3px` 테두리, `4px 4px 0 #111111` 하드 섀도우.
- 선택된 카드: 파란색 상단 바 `6px` 및 검정 테두리.
- 클릭 가능 카드 호버: 배경 `#FFF6B8`, 레이아웃 이동 없음.

### 9.6 테이블 및 목록

- 헤더 행: 검정 텍스트, 노란색 배경, `2px` 하단 테두리.
- 행 높이: 일반 `44px`, 밀집 `36px`.
- 선택된 행: 파란색 아웃라인과 연한 파란색 채움 `#E8F0FF`.
- 빈 테이블: 다음 CTA가 있는 중앙 카드.
- ID는 모노 칩으로 표시하며, 중간 잘림과 복사 동작을 제공한다.

### 9.7 상태 배너

배너 구조:

```txt
[아이콘] [상태 제목] [짧은 원인]                         [작업]
```

예시:

- `키링 사용 불가. API 키 저장 전에 시스템 SecretService 지원을 설치하세요.`

### 9.8 토큰 예산 바

- 높이: `18px`.
- 세그먼트: 시스템, 페르소나, 월드북, 로어북, 히스토리, 현재 메시지, 예약 출력.

### 9.11 채팅 메시지 버블

   - ✨ Vibe Fill 패널 (AI Persona×2 + Lorebook×2 + 바이브 + 카테고리 체크박스 + Provider/Model)
   - QProgressBar 진행률 표시
   - 생성 미리보기 체크리스트
   - 수동 엔트리 편집 폼

카테고리 (10개):

역사, 지리, 세력/국가, 종족, 마법/기술, 경제, 종교/신화, 던전/위험지대, 일상/문화, 규칙/법칙

## 13. 인터랙션 및 모션

### 일반 타이밍

| 인터랙션 | 타이밍 | 동작 |
|---|---:|---|
| 버튼 호버 | 120ms | 배경 및 섀도우 상태 변경 |
| 패널 열기/닫기 | 180ms | 너비 또는 불투명도 전환 |
| 토스트 진입/퇴장 | 180ms | 8px 슬라이드 + 페이드 |
| 스트리밍 커서 | 500ms 간격 | 스트리밍 중에만 깜빡임 |
| 드래그 재정렬 | 즉시 | 플레이스홀더 행이 포인터를 따름 |
| 검증 갱신 | blur 후 0ms, 텍스트 입력 중단 후 150ms | 인라인 오류 새로고침 |

### 키보드 동작

- `Ctrl/Cmd + N`: 채팅 페이지에서 새 채팅.
- `Ctrl/Cmd + Enter`: 입력창 포커스 시 세션이 활성 상태이면 전송.
- `Esc`: Inspector 서랍 또는 다이얼로그 닫기.
- `Ctrl/Cmd + S`: 주요 저장 CTA가 활성 상태일 때 현재 편집기 페이지 저장.
- 방향키로 목록 행 이동.
- 스페이스바로 선택된 체크박스/토글 전환.

### 로딩 상태

- Provider 연결 테스트는 `연결 테스트` 버튼에서만 차단 스피너를 표시한다.
- 모델 가져오기는 모델 캐시 패널에 진행률을 표시하고 교체 완료까지 기존 캐시를 유지한다.
- 스트리밍 어시스턴트 메시지는 이전 메시지를 이동하지 않고 청크를 추가한다.
- 폼 저장 시 현재 저장 CTA만 비활성화; 내비게이션은 사용 가능.

### 오류 상태

- 스트림 실패: 사용자 메시지 유지, 입력창 위에 빨간 배너 표시, 확인 후 세션을 `active`로 복귀.
- 키링 실패: Secret 패널에 빨간 배너 표시, 사용자가 페이지를 변경할 때까지 API 키 필드 내용 유지.
- Capability 가져오기 실패: 오류 배너 표시, 이전 캐시 테이블을 오래됨으로 표시.
- 저장 검증 실패: 첫 번째 유효하지 않은 필드에 포커스, 검증 패널에 모든 차단 오류 나열.

### 고정(Sticky) 동작

- 페이지 수준 저장 바는 편집기 패널 내부 하단에 고정.
- 채팅 입력창은 채팅 타임라인 하단에 고정.
- Inspector 헤더는 Inspector 스크롤 영역 내부에 고정.
- 상단 상태 바는 항상 표시.

### 축소 모션

OS 축소 모션 설정이 감지되면:

- 패널 전환이 즉시 실행.
- 토스트 슬라이드가 페이드만으로 변경.
- 스트리밍 커서는 표시되지만 깜빡이지 않음.

## 14. 접근성

### 최소 타겟

- 인터랙티브 타겟 크기: 최소 `36px`, 권장 `44px`.
- 포커스 링은 항상 표시되며 커스텀 스타일로 숨기지 않는다.
- 텍스트 입력 라벨은 모든 상태에서 표시를 유지한다.
- 필수 필드는 색상만이 아닌 텍스트와 아이콘을 사용한다.

### 스크린 리더 / 접근성 이름

모든 인터랙티브 웹 요소에 접근성 이름을 설정해야 한다:

```txt
Save Provider button
Test Connection button
Fetch Models button
Send Message button
Stop Streaming button
Show Prompt Snapshot button
Prompt Order Drag Handle: system_base
```

### 색상과 상태

- 상태 칩에 텍스트 라벨 포함: `연결됨`, `실패`, `스트리밍`, `중지됨`, `보관됨`.
- 오류 필드는 컨트롤 아래에 메시지 텍스트를 표시한다.
- 성공 상태는 체크 아이콘과 저장 타임스탬프를 포함한다.
- 경고는 적용된 기본값 또는 정확히 차단된 작업을 포함한다.

### 폼

- 탭 순서는 좌상단에서 우하단까지 시각적 순서를 따른다.
- 검증은 blur 시와 저장 시도 시 나타난다.
- 저장 차단 오류는 하단 검증 패널에 요약된다.
- 비밀번호 필드는 `secret_ref`만 복사 가능, 원시 API 키는 불가.

### 채팅 접근성

- 스트리밍 갱신은 모든 토큰 청크가 아닌 완료만 안내해야 한다.
- 메시지 버블은 역할, 타임스탬프, 토큰 추정을 노출한다.
- `중지`는 스트리밍 중 키보드로 접근 가능해야 한다.
- 프롬프트 Inspector 탭은 키보드 내비게이션을 사용한다.

- ModelProfile은 `supported_parameters`의 미지원 파라미터를 숨긴다.
- LM Studio 미확인 토큰 제한은 고정 기본값 경고를 표시한다.

## 16. 리스크 / 미결 사항

### 리스크

| 리스크 | 디자인 영향 | UI 대응 |
|---|---|---|
| Provider 메타데이터 형태 변경 | 파라미터 UI에 오래된 컨트롤 표시 가능 | capability 가져오기 타임스탬프와 원시 지원 파라미터 칩 표시 |
| LM Studio 토큰 제한 불가 | 토큰 예산이 부정확할 수 있음 | 고정 기본값 8192/2048의 지속적 노란색 경고 |
| 키링 백엔드 불가 | 사용자가 Gemini/OpenRouter 키를 저장할 수 없음 | 정확한 백엔드 요구사항이 있는 Secret 패널 오류 |
| 대용량 채팅 히스토리로 타임라인 느려짐 | UI가 무겁게 느껴질 수 있음 | 메시지 목록 가상화 또는 표시 메시지만 렌더링 |
| 프롬프트 미리보기 24000자 도달 | Inspector가 느려질 수 있음 | 잘린 미리보기 배너 표시, 원본 스냅샷 접근 유지 |
| 좁은 데스크톱 창 | 3컬럼 페이지가 비좁아짐 | 1120px 미만에서 컴팩트 레이아웃 규칙 사용 |

### 미결 사항

1. v0.1 BETA에서 `삭제` 작업 허용 여부가 정의되지 않았다. 데이터 삭제 정책이 지정될 때까지 영구 삭제 CTA를 노출하지 않아야 한다.
2. v0.1 BETA에 내보내기/가져오기 UI 포함 여부가 정의되지 않았다. 첫 번째 빌드에서 내보내기/가져오기 내비게이션 항목을 예약하지 않아야 한다.
3. 다크 모드 필요 여부가 정의되지 않았다. v0.1 BETA는 스타일링 및 QA 범위를 줄이기 위해 단일 라이트 테마로 출시한다.
4. 다중 세션 검색 필요 여부가 정의되지 않았다. 검색 동작이 지정된 후에만 세션 목록 필터링을 포함할 수 있다.
5. 메시지 간 프롬프트 스냅샷 비교 필요 여부가 정의되지 않았다. v0.1 BETA에서 Inspector는 한 번에 하나의 선택된 스냅샷만 표시한다.

```

    ├── 세션 제목 입력
    ├── ChatProfile 선택 (드롭다운)
    ├── UserPersona 선택 (드롭다운 or 자동 생성)

```

### 구현 시 주의사항

1. **캐시 버스팅**: 정적 파일 캐시 문제 방지를 위해 `?v=N` 쿼리 파라미터 사용
2. **z-index 레이어**: 모달은 `document.body` 직속 + `z-index: 9999`로 최상위 레이어 보장
3. **escapeHtml**: `providers.js`에 전역 정의, 모든 페이지에서 공유
4. **API 에러 처리**: `apiGet/apiPost` 실패 시 `try/catch`로 사용자 친화적 에러 메시지 표시
